How do I submit a package?

When you submit a mizOS package to the Duat, it is added to the community repository. This makes the package installable by users who's security level is set to "moderate" or lower. Otherwise, it can only be installed by users with the "none" security level set.


In order to submit a package, follow the steps below.



Step 1 - Basic Requirements

Your package must comply with the following:


1. It must be a package for the miz Operating System.


2. The package must not be available on the Arch Linux official repositories, or on the Asus-Linux repository.


3. Software installed by the package must have clear, well-written documentation.


4. The source code to the installed software must be available on any git website. (Github, Gitlab, Codeberg, etc.


5. Software installed by the package, and the package itself, must be completely void of any malware, spyware, horsefuckery, or any other bullshit.


6. The installed software must be licensed under any of the following licenses:

  • GNU GPL 3.0
  • GNU LGPL 3.0
  • MIT License
  • 3-Clause BSD License
  • 2-Clause BSD License
  • Apache License 2.0
  • WTFPL
  • DBAD License

Once you have verified that your package meets these requirements, you can proceed to the section below. Failure to comply with any of the terms listed above will restrict your package from being added to the community repository.



Step 2 - Submitting the Package

Package submissions are done through the Discord server. Please include the following information in your submission:

  • Name of the package. (user/repo)
  • Name of the installed software.
  • The installed software's license.
  • A link to the package's Github repository.
  • A link to the installed software's source code. (git repository)
  • A personal identification link to the software's main developer.
  • A personal identification link to the package's main maintainer.
  • A short description of the installed software. (25 words max)
